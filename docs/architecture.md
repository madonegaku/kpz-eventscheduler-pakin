# Архітектурний опис проєкту: Pakin Event Scheduler

Цей документ описує доменну область, ключових акторів та технічні межі системи планування подій.

## 1. Проблема, яку вирішує система

Система **Pakin Event Scheduler** розроблена для вирішення проблеми неефективного управління часом та труднощів у координації спільних заходів.

**Ключові виклики:**

- **Накладання подій:** Користувачі часто призначають зустрічі на один і той самий час.
- **Складність координації:** Важко відстежувати, хто з запрошених учасників підтвердив присутність.
- **Відсутність нагадувань:** Ризик пропустити важливу подію через відсутність автоматичних сповіщень.

## 2. Користувачі (Ролі)

У системі передбачено 5 основних ролей, визначених на етапі аналізу вимог:

1. **Гість (Guest):** Неавторизований відвідувач. Може переглядати лендінг та реєструватися.
2. **Користувач (User):** Авторизована особа. Має доступ до особистого календаря.
3. **Організатор (Organizer):** Користувач, який створює подію та запрошує інших.
4. **Учасник (Attendee):** Запрошена особа, яка може прийняти або відхилити запит на участь.
5. **Адміністратор (Admin):** Відповідає за технічну підтримку та модерацію системи.

## 3. Основні операції системи

Система реалізує наступні ключові бізнес-процеси:

- **Автентифікація:** Реєстрація та безпечний вхід за допомогою email/пароля.
- **Управління подіями (CRUD):** Створення, перегляд списку, редагування деталей та видалення подій.
- **Перевірка конфліктів:** Автоматичне виявлення перетину часових інтервалів при плануванні.
- **Управління запрошеннями:** Надсилання запитів іншим користувачам системи.
- **RSVP-менеджмент:** Можливість для учасників підтвердити або відхилити запрошення.
- **Система сповіщень:** Надсилання нагадувань перед початком запланованих заходів.
- **Категоризація:** Присвоєння тегів (напр., «Робота», «Особисте») для фільтрації розкладу.

## 4. Зовнішні системи

Для забезпечення стабільної роботи система інтегрується з такими ресурсами:

| Система                            | Призначення                                                  |
| :--------------------------------- | :----------------------------------------------------------- |
| **База даних (PostgreSQL)**        | Збереження профілів, логів та всіх даних про події.          |
| **SMTP Сервіс (SendGrid/Mailgun)** | Надсилання сервісних листів та запрошень на email.           |
| **Firebase Cloud Messaging**       | Надсилання push-сповіщень на браузер або мобільні пристрої.  |
| **External Calendars (Optional)**  | Синхронізація з Google Calendar або iCal через офіційні API. |
